# 와이어프레임 (Wireframe) — 페이지별 상세 설계

> PRD.md 기준. 모든 페이지는 공통 컴포넌트(Header/Footer/플로팅 버튼)를 포함하며, 아래 본문에서는 페이지별 메인 콘텐츠 영역만 다룬다.

---

# 공통 컴포넌트 (전 페이지 적용)

## Header (GNB)
**포함 요소**
- 로고 (좌측, 클릭 시 `/` 이동, 관리자 이미지 교체 대상)
- 메인 메뉴: 회사소개(드롭다운: 인사말/회사개요) / 사업분야 / 시공사례 / 보유역량(선택) / 고객문의
- 우측: 전화번호(클릭콜 `tel:`) + "문의하기" 버튼(`/contact`)
- 스크롤 시 sticky 고정, 높이 축소(예: 80px → 60px)
- Mobile: 로고(좌) + 햄버거 아이콘(우) → 풀스크린 슬라이드 메뉴, 메뉴 하단 고정 "문의하기" 버튼

**사용자 행동 유도 목적**
- 페이지 어디서든 1클릭 이내로 문의 페이지 또는 핵심 메뉴에 접근 가능하게 하여 이탈률 감소

## Footer
**포함 요소**
- 사업자정보: 상호 / 대표자 / 사업자등록번호 / 주소 / 전화 / 이메일
- 정책 링크: 개인정보처리방침, 이용약관
- SNS 링크(선택)
- Copyright 텍스트
- Mobile: 아코디언으로 정보 그룹 축약 가능

**사용자 행동 유도 목적**
- 법적 신뢰 정보를 상시 노출하여 거래 신뢰도 보강(직접 전환 유도보다는 신뢰 보강 목적)

## 플로팅 버튼 (Mobile 우선, 전 페이지 공통)
**포함 요소**
- 우측 하단 고정: "전화문의"(`tel:`) + "빠른문의"(`/contact` 또는 문의 모달)
- 스크롤 300px 이상 시 노출, "맨 위로" 버튼 동반

**사용자 행동 유도 목적**
- 모바일 사용자가 페이지 중간 어디서든 즉시 전환 행동을 취할 수 있게 함(스크롤 후 헤더 재접근 마찰 제거)

# 1. 메인(홈)

**URL:** `/`
**섹션 순서:** 1) Hero 2) 핵심 강점 요약(USP) 3) 사업분야 미리보기 4) 시공사례 하이라이트 5) 회사소개 요약/신뢰지표 6) 문의 유도 CTA 7) Footer(공통)

## 1. Hero Section
**포함 요소**
- 배경: 메인 비주얼 이미지 슬라이더(관리자 교체 가능, 최대 5장, autoplay 5초, dot indicator)
- 제목(H1): 회사 슬로건 (예: "30년 노하우, 신뢰의 시공")
- 부제목: 회사 소개 한줄 문구(보유 자료 "회사 소개 문구" 활용)
- CTA 버튼 2개: Primary "무료 견적 문의"(`/contact`), Secondary "시공사례 보기"(`/works`)
- Mobile: 이미지 세로 비율로 별도 크롭, 텍스트 중앙정렬, 버튼은 풀와이드 세로 스택

**사용자 행동 유도 목적**
- 첫 화면에서 신뢰감과 핵심 행동(문의/사례확인)을 동시에 제시해 스크롤 또는 즉시 전환 중 하나로 분기

## 2. 핵심 강점 요약(USP) 섹션
**포함 요소**
- 아이콘 + 숫자 카드 4개 (예: 시공실적 N건 / 보유장비 N대 / 보유인증 N개 / 업력 N년)
- Desktop 4열 그리드 / Mobile 2열 그리드

**사용자 행동 유도 목적**
- 수치 기반 신뢰 지표로 체류 유도 및 하단 콘텐츠로 스크롤 유도

## 3. 사업분야 미리보기 섹션
**포함 요소**
- 섹션 타이틀 "사업분야"
- 카드 3~4개(아이콘/이미지 + 분야명 + 한줄설명), 카드 클릭 시 `/business`
- Mobile: 가로 스와이프 카드 또는 1열 스택

**사용자 행동 유도 목적**
- 서비스 범위를 빠르게 인지시키고 관심 분야의 세부 페이지로 유입

## 4. 시공사례 하이라이트 섹션
**포함 요소**
- 섹션 타이틀 "시공사례" + 우측 "전체보기 →" 링크(`/works`)
- 카드 3~6개(썸네일 + 카테고리 뱃지 + 제목), hover 시 확대 효과
- Mobile: 가로 스와이프 캐러셀

**사용자 행동 유도 목적**
- 실제 시공 결과물 노출로 신뢰 형성, `/works` 상세 콘텐츠 유입 유도

## 5. 회사소개 요약/신뢰지표 섹션
**포함 요소**
- 좌: 회사 소개 문구 요약(2~3문단) + "회사소개 더보기" 버튼(`/about`)
- 우: 사업자등록증 썸네일 또는 인증마크 이미지(클릭 시 라이트박스 확대)
- Mobile: 텍스트 우선 배치, 이미지는 하단

**사용자 행동 유도 목적**
- 법적 실체(사업자등록증)를 시각적으로 확인시켜 신뢰도를 즉시 검증

## 6. 문의 유도 CTA 섹션
**포함 요소**
- Full-width 강조 배경 배너
- 문구: "지금 바로 무료 견적을 받아보세요"
- 전화번호(클릭콜) + "온라인 문의하기" 버튼(`/contact`)

**사용자 행동 유도 목적**
- 페이지 최하단 이탈 직전 시점에 마지막 전환 기회를 제공

# 2. 회사소개 - 인사말/비전

**URL:** `/about`
**섹션 순서:** 1) 서브 히어로(Breadcrumb) 2) 회사소개 하위 탭 내비게이션 3) 대표 인사말 4) 비전/미션/핵심가치 5) 연혁 타임라인(선택) 6) 문의 유도 CTA

## 1. 서브 히어로
**포함 요소**
- 작은 배경 이미지 + H1 "회사소개" + Breadcrumb(홈 > 회사소개)

**사용자 행동 유도 목적**
- 현재 위치 인지 및 사이트 구조 이해를 도와 탐색 이탈 방지

## 2. 하위 탭 내비게이션
**포함 요소**
- 탭: "인사말"(활성) | "회사개요"(`/about/company`)

**사용자 행동 유도 목적**
- 회사소개 하위 페이지 간 이동 마찰을 최소화하여 추가 신뢰 정보(사업자등록증 등) 열람 유도

## 3. 대표 인사말 섹션
**포함 요소**
- 좌: 대표이미지(선택) / 우: 인사말 텍스트 3~5문단 + 서명 이미지

**사용자 행동 유도 목적**
- 경영진의 메시지를 통해 인간적 신뢰와 책임감을 전달

## 4. 비전/미션/핵심가치 섹션
**포함 요소**
- 카드 3개: 비전 / 미션 / 핵심가치 (아이콘 + 제목 + 설명)

**사용자 행동 유도 목적**
- 회사의 방향성과 전문성을 인지시켜 신뢰도 보강

## 5. 연혁 타임라인 (선택)
**포함 요소**
- Desktop: 가로 타임라인 / Mobile: 세로 타임라인, 연도별 주요 이벤트

**사용자 행동 유도 목적**
- 업력과 성장 스토리로 안정성 인식 강화

## 6. 문의 유도 CTA
**포함 요소**
- 공통 CTA 패턴(전화번호 + "문의하기" 버튼)

**사용자 행동 유도 목적**
- 회사 신뢰 확인 직후 자연스러운 전환 동선 제공

# 3. 회사소개 - 회사개요

**URL:** `/about/company`
**섹션 순서:** 1) 서브 히어로 + 탭 2) 회사개요 표 3) 사업자등록증 4) 조직도/인력현황(선택) 5) 위치 요약 카드 6) 문의 유도 CTA

## 1. 서브 히어로 + 하위 탭
**포함 요소**
- Breadcrumb(홈 > 회사소개 > 회사개요), 탭(인사말 | 회사개요(활성))

**사용자 행동 유도 목적**
- 위치 인지 및 신속한 탭 전환

## 2. 회사개요 표 섹션
**포함 요소**
- Key-Value 2열 표: 상호 / 대표자 / 설립일 / 주소 / 사업영역 / 전화 / 이메일 / 사업자등록번호
- Mobile: 표를 1열 스택 형태로 변경(라벨-값 세로 배치)

**사용자 행동 유도 목적**
- 법인 정보를 투명하게 공개하여 B2B 거래 신뢰 확보

## 3. 사업자등록증 섹션
**포함 요소**
- 사업자등록증 이미지 썸네일(관리자 업로드, 필요 시 일부 정보 마스킹) + "확대보기" 버튼(라이트박스 모달)
- 캡션: 사업자등록번호 텍스트 병기

**사용자 행동 유도 목적**
- 실존 사업자임을 시각적으로 증명하는 핵심 신뢰 요소(특히 B2B/공공 발주처 대상)

## 4. 조직도/인력현황 (선택)
**포함 요소**
- 부서별 카드 나열 또는 간단한 조직도 이미지

**사용자 행동 유도 목적**
- 회사 규모와 체계를 전달하여 대형 프로젝트 수행 역량 신뢰도 보강

## 5. 위치 요약 카드
**포함 요소**
- 지도 썸네일(클릭 시 `/contact#location` 이동) + 주소 텍스트

**사용자 행동 유도 목적**
- 오프라인 실재 사업장을 확인시키고 문의 페이지로 자연스럽게 연결

## 6. 문의 유도 CTA
**포함 요소**
- 공통 CTA 패턴

**사용자 행동 유도 목적**
- 회사 정보 확인 후 즉시 행동 전환

# 4. 사업분야

**URL:** `/business`
**섹션 순서:** 1) 서브 히어로 2) 사업분야 개요 인트로 3) 서비스 항목별 상세(반복 블록) 4) 진행 프로세스 다이어그램 5) 문의 유도 CTA

## 1. 서브 히어로
**포함 요소**
- Breadcrumb(홈 > 사업분야) + H1 "사업분야"

**사용자 행동 유도 목적**
- 위치 인지

## 2. 사업분야 개요 인트로
**포함 요소**
- 1~2문단 소개 텍스트(전체 사업 범위 요약)

**사용자 행동 유도 목적**
- 세부 항목을 읽기 전 전체 그림 제공으로 이해도 향상

## 3. 서비스 항목별 상세 섹션 (반복 블록, N개)
**포함 요소**
- 이미지(좌/우 교차 배치) + 항목 제목 + 설명 텍스트 + "관련 시공사례 보기" 링크(`/works?category=xxx`)
- Mobile: 이미지 상단, 텍스트 하단으로 통일(교차배치 제거)

**사용자 행동 유도 목적**
- 서비스별 전문성을 구체적으로 전달하고, 카테고리 필터링된 시공사례로 자연스럽게 교차 유입

## 4. 진행 프로세스 다이어그램 섹션
**포함 요소**
- 가로 스텝형 다이어그램: 수주 → 설계/계획 → 시공 → 품질검사 → 준공/AS
- Mobile: 세로 스텝형으로 전환

**사용자 행동 유도 목적**
- 작업 체계성을 시각화하여 "이 회사가 일을 어떻게 진행하는지" 막연한 불안감 해소

## 5. 문의 유도 CTA
**포함 요소**
- 문구: "내 프로젝트에 맞는 견적이 궁금하신가요?" + "견적 문의하기" 버튼(`/contact`)

**사용자 행동 유도 목적**
- 서비스 이해 직후 구체적 견적 문의로 전환

# 5. 시공사례 목록

**URL:** `/works`
**섹션 순서:** 1) 서브 히어로 2) 필터/검색 바 3) 카드 그리드 4) 페이지네이션 5) 문의 유도 CTA

## 1. 서브 히어로
**포함 요소**
- Breadcrumb(홈 > 시공사례) + H1 "시공사례" + 한줄 설명

**사용자 행동 유도 목적**
- 위치 인지 및 페이지 목적 명확화

## 2. 필터/검색 바
**포함 요소**
- 카테고리 탭(전체/공장/플랜트/리모델링/토목 등, 관리자가 등록한 카테고리 동적 반영)
- 정렬 옵션(최신순 기본), 선택적 지역/연도 드롭다운
- Mobile: 가로 스크롤 탭 + 필터 아이콘 클릭 시 바텀시트로 상세 필터 노출

**사용자 행동 유도 목적**
- 관심 분야로 빠르게 좁혀 탐색 효율을 높이고 체류시간 증가

## 3. 카드 그리드 섹션
**포함 요소**
- Desktop 3~4열 / Tablet 2열 / Mobile 1~2열
- 카드 구성: 썸네일 + 카테고리 뱃지 + 제목 + 간단설명 + 등록일
- Hover/Tap 시 "자세히보기" 오버레이

**사용자 행동 유도 목적**
- 시공 역량을 비주얼로 증명하고 상세 페이지 클릭을 유도

## 4. 페이지네이션
**포함 요소**
- Desktop: 번호형 페이지네이션 / Mobile: "더보기" 버튼 또는 무한스크롤

**사용자 행동 유도 목적**
- 콘텐츠 과다 로딩으로 인한 속도 저하·이탈 방지

## 5. 문의 유도 CTA
**포함 요소**
- 문구: "찾는 시공 사례가 없으신가요? 직접 문의해보세요" + 버튼(`/contact`)

**사용자 행동 유도 목적**
- 목록 탐색 후에도 원하는 사례를 못 찾은 사용자를 문의로 전환

# 6. 시공사례 상세

**URL:** `/works/:id`
**섹션 순서:** 1) Breadcrumb + 타이틀 2) 메인 이미지 갤러리 3) 프로젝트 정보 요약 4) 작업 설명 본문 5) 이전/다음 + 관련 사례 6) 문의 유도 CTA(하단 고정 가능)

## 1. Breadcrumb + 타이틀 섹션
**포함 요소**
- Breadcrumb(홈 > 시공사례 > {카테고리} > {제목})
- H1 제목 + 카테고리 뱃지

**사용자 행동 유도 목적**

# 8. 고객문의

**URL:** `/contact`
**섹션 순서:** 1) 서브 히어로 2) 연락처 정보 카드 3) 문의폼 4) 지도/오시는 길 5) 제출 완료 안내(상태 기반)

## 1. 서브 히어로
**포함 요소**
- Breadcrumb(홈 > 고객문의) + H1 "고객문의" + 안내 문구("궁금한 점을 남겨주시면 빠르게 연락드립니다")

**사용자 행동 유도 목적**
- 문의 행위에 대한 기대치(응답속도 등)를 명시해 작성 의지 강화

## 2. 연락처 정보 카드 섹션
**포함 요소**
- 아이콘 + 텍스트 카드 3~4개: 전화(`tel:` 클릭콜) / 이메일(`mailto:`) / 주소 / 운영시간

**사용자 행동 유도 목적**
- 폼 작성이 부담스러운 사용자에게 즉시 채널(전화) 제공 → 전환 경로 다양화

## 3. 문의폼 섹션
**포함 요소**
- 필드: 이름*(text) / 연락처*(tel) / 이메일*(email) / 문의유형(select: 견적·제휴·AS·기타) / 제목(text) / 내용*(textarea) / 첨부파일(선택, 최대 5MB, jpg·png·pdf) / 개인정보 수집·이용 동의*(checkbox)
- 실시간 유효성 검사: 필수값 미입력 시 인라인 에러 메시지, 이메일/전화번호 형식 검증
- 스팸 방지: reCAPTCHA 또는 honeypot 히든필드
- 제출 버튼: "문의 보내기"(제출 중 로딩 스피너 + 중복클릭 방지), 동의 체크 전 버튼 비활성화
- Mobile: 필드 전체 풀와이드, 셀렉트는 네이티브 picker 활용

**사용자 행동 유도 목적**
- 입력 마찰을 최소화한 폼 구조로 작성 중도 이탈률 감소, 정확한 리드 정보 확보 → 문의 전환율 핵심 구간

## 4. 지도/오시는 길 섹션
**포함 요소**
- 지도 임베드(Kakao Map/Google Map) + 주소 텍스트 + 대중교통/주차 안내

**사용자 행동 유도 목적**
- 실사업장 존재를 보여줘 신뢰 보강 및 현장 방문 가능성 제고

## 5. 제출 완료 안내
**포함 요소**
- 인라인 메시지 또는 모달: "문의가 정상적으로 접수되었습니다. 빠른 시간 내 연락드리겠습니다."
- 실패 시: 에러 메시지 + 재시도 버튼 + 직접 전화 안내 문구

**사용자 행동 유도 목적**
- 접수 확인으로 사용자 불안 해소, 실패 시에도 대체 전환 경로(전화) 즉시 제시
