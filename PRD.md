# 제조·건설업 회사 홈페이지 PRD (Product Requirements Document)

## 0. 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 제조·건설업 회사 홈페이지 구축 |
| 목적 | 회사 신뢰도 확보, 시공 사례 홍보, 문의 전환 증가 |
| 규모 | 약 10페이지 |
| 대응 | 반응형 웹 (모바일 최적화 우선) |
| 핵심 기능 | 관리자 로그인, 이미지 교체(CMS형), 문의폼, 문의 시 회사 메일 자동 발송 |
| 보유 자료 | 로고, 시공사례 이미지, 메인 비주얼 이미지, 사업자등록증, 회사 소개 문구 |

### 0-1. 핵심 목표와 우선순위
1. **신뢰도 확보** — 사업자등록증·연혁·시공실적 등 객관적 근거를 통해 "검증된 회사"라는 인상 전달
2. **시공 사례 홍보** — 카테고리/연도별로 정리된 포트폴리오로 실제 작업 역량을 시각적으로 증명
3. **문의 전환 증가** — 모든 페이지에서 문의 진입을 막지 않고, 문의 작성 마찰을 최소화

## 1. 사이트맵

```
HOME (메인)
├── 회사소개
│   ├── 인사말 / 비전
│   ├── 회사개요 (사업자등록증, 연혁, 조직)
│   └── 오시는 길
├── 사업분야 (제조/건설 서비스 소개)
├── 시공사례 (포트폴리오 목록)
│   └── 시공사례 상세 (개별 케이스)
├── 보유역량 (설비/장비/인증/자격증) *선택
├── 고객문의 (문의폼 + 오시는 길/연락처)
├── 공지사항 / 소식 *선택(간단 게시판)
└── (Footer 공통) 개인정보처리방침, 이용약관

[관리자 영역 - 비공개]
└── /admin
    ├── 로그인
    ├── 대시보드
    ├── 이미지 관리 (메인비주얼/시공사례)
    ├── 시공사례 게시물 관리 (등록/수정/삭제)
    └── 문의내역 관리
```

### 1-1. 페이지 목록 (10페이지 기준)

| No | 페이지 | URL(예시) | 비고 |
|---|---|---|---|
| 1 | 메인(홈) | `/` | 메인 비주얼, 핵심 요약 |
| 2 | 회사소개 - 인사말/비전 | `/about` | |
| 3 | 회사소개 - 회사개요 | `/about/company` | 사업자등록증 노출 |
| 4 | 사업분야 | `/business` | |
| 5 | 시공사례 목록 | `/works` | 필터/카테고리 |
| 6 | 시공사례 상세 | `/works/:id` | 동적 페이지(템플릿 1개로 다수 콘텐츠 처리) |
| 7 | 보유역량(설비/인증) | `/capability` | 선택사항, 없으면 회사소개에 통합 |
| 8 | 고객문의 | `/contact` | 문의폼 핵심 페이지 |
| 9 | 오시는 길 | `/contact#location` 또는 `/location` | 지도 포함, 문의 페이지에 통합 가능 |
| 10 | 관리자 로그인/대시보드 | `/admin` | 비공개, 검색엔진 제외 |

> 시공사례 상세는 템플릿 1개로 여러 게시물을 처리하므로 "페이지 수"에는 1개로 카운트. 총 9개 공개 페이지 + 관리자 1세트로 약 10페이지 규모 충족.

## 2. 페이지별 기능

### 2-1. 메인(홈) `/`
- **메인 비주얼 슬라이더/배너**: 관리자가 이미지 교체 가능, 1~5장 권장, 모바일에서는 세로 비율 별도 이미지 또는 자동 크롭
- 회사 한줄 소개 문구 (회사 소개 문구 자료 활용)
- 핵심 강점 요약 (3~4개 아이콘+텍스트: 시공실적, 보유장비, 인증, 연혁 등)
- 대표 시공사례 하이라이트 (최신 3~6건, 시공사례 목록과 연동)
- 빠른 문의 CTA 배너 (전화번호 클릭콜, "견적문의" 버튼)
- Footer: 사업자정보, 주소, 연락처, 개인정보처리방침 링크

### 2-2. 회사소개 - 인사말/비전 `/about`
- 대표 인사말 (회사 소개 문구 활용)
- 비전/미션/핵심가치
- 연혁 타임라인 (선택)

### 2-3. 회사소개 - 회사개요 `/about/company`
- 회사 개요 표 (상호, 대표자, 설립일, 주소, 사업영역, 연락처)
- **사업자등록증 이미지 노출** (신뢰도 핵심 요소, 워터마크 또는 등록번호 일부 마스킹 검토)
- 조직도/인력 현황 (선택)

### 2-4. 사업분야 `/business`
- 제조/건설 서비스 항목별 소개 (텍스트 + 이미지)
- 서비스 프로세스(수주→설계→시공→준공) 다이어그램
- 각 항목에서 "문의하기" CTA 연결

### 2-5. 시공사례 목록 `/works`
- 카테고리 필터 (예: 공장/플랜트/리모델링/토목 등, 운영 중 추가 가능해야 함)
- 연도/지역 필터 (선택)
- 카드형 그리드 레이아웃 (썸네일 이미지 + 제목 + 간단 정보)
- 페이지네이션 또는 무한스크롤
- 정렬: 최신순 기본

### 2-6. 시공사례 상세 `/works/:id`
- 대표 이미지 + 갤러리(다중 이미지, 슬라이더)
- 프로젝트 정보 (발주처/규모/공사기간/위치 — 비공개 요청 항목은 표시 옵션 처리)
- 작업 설명 텍스트
- 이전/다음 사례 네비게이션
- 하단 문의 유도 CTA

### 2-7. 보유역량 `/capability` (선택)
- 보유 장비/설비 이미지 목록
- 인증서/자격증 이미지 목록
- 회사개요 페이지에 통합해도 무방 (페이지 수 절감 시)

### 2-8. 고객문의 `/contact`
- **문의폼**: 이름, 연락처(필수), 이메일(필수), 문의유형(선택: 견적/제휴/AS/기타), 제목, 내용, 첨부파일(선택, 도면/사진 등), 개인정보 수집 동의 체크박스(필수)
- 폼 유효성 검사 (필수값, 전화번호/이메일 형식)
- 스팸 방지 (reCAPTCHA 또는 honeypot)
- 제출 성공/실패 안내 (토스트 또는 완료 페이지)
- 회사 연락처 정보(전화, 이메일, 팩스), 카카오톡채널/네이버톡톡 등 선택 채널 노출
- 지도(Google Maps/Kakao Map 임베드) + 주소 + 오시는 길 안내

### 2-9. 오시는 길
- `/contact` 내 섹션으로 통합 (별도 페이지 미생성 권장 → 페이지 수 절감 및 UX 단순화)

### 공통 컴포넌트
- Header: 로고, GNB(메뉴), 모바일 햄버거 메뉴, 전화 클릭콜 버튼
- Footer: 사업자정보, 주소, 대표/사업자등록번호, 개인정보처리방침/이용약관 링크, SNS 링크(선택)
- 플로팅 버튼: 우측 하단 "빠른 문의" / "맨 위로" 버튼 (모바일 전환 유도용)

## 3. 관리자 기능

### 3-1. 관리자 로그인 `/admin/login`
- ID/PW 기반 로그인 (단일 또는 소수 관리자 계정)
- 비밀번호 해시 저장(bcrypt 등), HTTPS 필수
- 로그인 실패 횟수 제한 (브루트포스 방지)
- 세션 만료 처리 (예: 1~2시간 비활동 시 자동 로그아웃)
- 비밀번호 재설정 (이메일 기반, 선택)

### 3-2. 관리자 대시보드 `/admin`
- 최근 문의 내역 요약 (미확인 건수 표시)
- 시공사례 게시물 수, 최근 등록일 등 요약 통계

### 3-3. 이미지 관리 기능 (핵심 요구사항)
- **메인 비주얼 이미지 교체**: 업로드/삭제/순서변경(드래그앤드롭), 권장 사이즈 가이드 제공
- **로고 교체**: 헤더/푸터 로고 업로드
- **시공사례 이미지 관리**: 게시물별 다중 이미지 업로드, 대표 이미지 지정, 갤러리 순서 조정
- 이미지 업로드 시 자동 리사이즈/최적화(WebP 변환, 용량 제한 예: 5MB) 처리
- 업로드 가능 포맷 제한 (jpg, png, webp) 및 검증 (확장자+MIME 타입 동시 체크)

### 3-4. 시공사례 게시물 관리
- 등록/수정/삭제(CRUD)
- 카테고리 추가/관리
- 노출 여부(공개/비공개) 토글, 게시 순서(고정/정렬) 설정

### 3-5. 문의내역 관리
- 문의 목록 조회 (날짜/상태별 필터: 신규/처리중/완료)
- 상세 내용 확인, 처리 상태 변경
- 첨부파일 다운로드
- CSV 내보내기 (선택)
- 개인정보 보관기간 정책에 따른 자동/수동 삭제 기능 (개인정보보호법 대응)

### 3-6. 권한 및 보안
- 관리자 페이지는 `robots.txt`로 검색엔진 색인 제외
- 모든 관리자 API는 인증 미들웨어로 보호
- 관리자 활동 로그 기록 (선택, 누가 언제 무엇을 수정했는지)

## 4. 문의 프로세스

### 4-1. 사용자 플로우
1. 사용자가 `/contact` 페이지 또는 사이트 내 CTA(시공사례 상세, 메인 배너 등)를 통해 문의폼 진입
2. 필수 항목 입력 (이름, 연락처, 이메일, 문의내용) + 개인정보 수집 동의
3. 스팸 검증(reCAPTCHA) 통과
4. "제출" 클릭 → 서버에서 유효성 재검증
5. DB에 문의 데이터 저장 (관리자 페이지에서 조회 가능하도록)
6. **회사 담당 메일주소로 자동 발송** (제목: [홈페이지 문의] + 문의유형/이름, 본문: 입력 내용 전체)
7. 사용자에게 접수 확인 메일 발송 (선택, 신뢰도 향상에 도움) 또는 화면 내 "정상 접수되었습니다" 안내
8. 관리자는 대시보드에서 신규 문의 확인 → 상태를 "처리중/완료"로 변경하며 관리

### 4-2. 기술 처리 흐름
```
[사용자 문의폼 제출]
   → Frontend 유효성 검사
   → Backend API 호출 (POST /api/contact)
       → 서버 측 유효성/스팸 검증
       → DB 저장 (Contacts 테이블)
       → 메일 발송 서비스 호출 (예: SMTP, SendGrid, Nodemailer 등)
           → 회사 메일함으로 발송 (필수)
           → 사용자 확인 메일 발송 (선택)
       → 발송 실패 시에도 DB 저장은 유지, 관리자 페이지에서 확인 가능(메일 발송 실패가 문의 자체 누락으로 이어지지 않도록)
   → 사용자에게 성공/실패 응답 반환
```

### 4-3. 예외 처리 요구사항
- 메일 발송 실패 시에도 DB에는 반드시 저장 (이중 안전장치)
- 발송 실패 시 관리자에게 별도 알림(재시도 큐 또는 에러 로그) 권장
- 동일 사용자의 반복 제출(스팸/중복) 방지: 짧은 시간 내 중복 제출 차단

### 4-4. 개인정보 처리
- 문의폼 하단에 "개인정보 수집 및 이용 동의" 명시, 미동의 시 제출 불가
- 개인정보처리방침 페이지(Footer 링크) 별도 제공: 수집 항목, 이용 목적, 보관 기간, 파기 절차 명시
- 수집 항목은 문의 처리에 필요한 최소한으로 제한 (이름, 연락처, 이메일, 문의내용)

## 5. SEO 요구사항

### 5-1. 기본 SEO
- 모든 페이지 고유 `title`, `meta description` 설정 (회사명 + 핵심 키워드 조합, 예: "OOO건설 | 공장/플랜트 시공 전문")
- `canonical` 태그 적용 (중복 URL 방지)
- `sitemap.xml` 자동 생성 및 제출 (Google Search Console, 네이버 서치어드바이저)
- `robots.txt` 설정: 공개 페이지는 색인 허용, `/admin` 등 관리자 경로는 색인 차단
- 시맨틱 HTML 마크업 (h1은 페이지당 1개, 논리적 heading 구조)

### 5-2. 콘텐츠 SEO
- 지역 기반 키워드 반영 (예: "OO지역 공장 시공", "OO 플랜트 건설") — 지역 검색 노출 강화
- 시공사례 상세 페이지마다 고유한 title/description 자동 생성 (사례명, 카테고리 기반)
- 이미지 `alt` 텍스트 필수 입력 (시공사례 이미지 등 — 관리자 업로드 시 alt 입력 필드 제공)
- 본문 내 자연스러운 키워드 배치 (제조/건설 업종 + 서비스명 + 지역명)

### 5-3. 구조화 데이터 (Schema.org)
- `LocalBusiness` 또는 `Organization` 스키마 적용 (회사명, 주소, 전화번호, 영업시간, 로고)
- `BreadcrumbList` 스키마 적용 (특히 시공사례 상세 페이지)
- 가능 시 `Article`/`CreativeWork` 스키마로 시공사례 콘텐츠 보강

### 5-4. 기술적 SEO (성능/모바일)
- Core Web Vitals 기준 충족 목표 (LCP 2.5초 이하, CLS 0.1 이하)
- 이미지 최적화: WebP 변환, lazy loading 적용 (특히 시공사례 갤러리)
- 모바일 우선 반응형 레이아웃 (Google Mobile-Friendly 기준 충족)
- HTTPS 적용 필수
- 페이지 로딩 속도 최적화 (이미지 CDN 또는 압축, 코드 스플리팅)

### 5-5. 외부 연동
- Google Search Console / 네이버 서치어드바이저 등록
- Google Analytics(GA4) 또는 유사 분석 툴 설치 — 문의 전환 추적(목표: 문의폼 제출 이벤트 트래킹)
