# 디자인 시스템 (Design System)

> 타겟: 제조·건설업 B2B 의사결정자(40~60대 대표). 핵심 인상 목표: **신뢰감 / 전문성 / 안정감**
> 기술 기준: Tailwind CSS (`TECH_STACK.md`, `ARCHITECTURE.md`의 `tailwind.config.ts`에 그대로 적용 가능한 값으로 작성)

## 0. 디자인 원칙 (모든 항목에 공통 적용)

| 원칙                          | 내용                                                            | 타겟 근거                                                                 |
| ----------------------------- | --------------------------------------------------------------- | ------------------------------------------------------------------------- |
| 차분한 보색 대비              | 네이비(신뢰) + 차분한 오렌지(행동유도) 2색 중심, 다색 사용 지양 | 40~60대는 화려한 그라데이션/네온 컬러보다 절제된 배색을 "전문적"으로 인식 |
| 충분히 큰 글자/터치영역       | 본문 최소 16px, 터치 타겟 최소 44px                             | 시인성·조작 정확도가 전환율(문의 클릭)에 직결                             |
| 과한 모션 지양                | 모든 애니메이션은 150~300ms, 의미 있는 곳에만 적용              | 화려한 모션은 "스타트업 느낌"으로 오히려 안정감을 해침                    |
| 여백으로 신뢰를 만든다        | 섹션 간 여백을 넉넉히 확보, 콘텐츠를 빽빽하게 채우지 않음       | 여백이 부족하면 저가/영세 이미지로 인식될 위험                            |
| 데이터·근거를 시각적으로 강조 | 사업자등록증, 인증서, 수치(시공실적 등)는 카드/표로 명확히 구획 | 신뢰는 "느낌"이 아니라 "근거 제시"에서 나옴                               |

| `brand-700` (Primary)      | `#16304D` | Header 배경(선택), 헤딩 텍스트, 강조 보더       | 네이비는 금융·건설·제조 업종에서 가장 보편적인 "신뢰" 컬러                              |

| `accent-600` (CTA)         | `#D9480F` | "문의하기" 등 전환 버튼, 핵심 강조 포인트       | 건설 안전색(시그널 오렌지) 계열 — 업종 정체성 + 네이비와 강한 보색 대비로 CTA 시선 집중 |
| `accent-500`               | `#E8590C` | 버튼 hover 전 기본 상태(밝은 쪽), 아이콘 포인트 |                                                                                         |

| 토큰명        | HEX       | 용도                                       |
| ------------- | --------- | ------------------------------------------ |
| `neutral-50`  | `#F7F8FA` | 섹션 배경(흰색과 교차 사용, 단조로움 방지) |
| `neutral-100` | `#EEF0F3` | 구분선 배경, 비활성 영역                   |
| `neutral-200` | `#DDE1E6` | 카드 보더, 입력창 보더                     |
| `neutral-400` | `#9AA3AE` | placeholder, 비활성 텍스트                 |
| `neutral-600` | `#5A6472` | 보조 텍스트(설명문)                        |

### 1-3. 시맨틱 컬러 (상태 표시 — 관리자 화면 중심)

| 토큰명            | HEX              | 용도                      |
| ----------------- | ---------------- | ------------------------- |
| `status-new`      | `#2563EB` (블루) | 문의 상태: 신규           |
| `status-progress` | `#D97706` (앰버) | 문의 상태: 처리중         |
| `status-done`     | `#16A34A` (그린) | 문의 상태: 완료           |
| `status-error`    | `#DC2626` (레드) | 폼 유효성 에러, 삭제 경고 |

### 1-4. 사용 규칙

- 한 화면에서 `accent-600`(오렌지)은 **CTA 버튼 1~2개 이내**로만 사용한다(남용 시 시선 분산, 전환 효과 저하).
- 배경은 `white`와 `neutral-50`을 섹션 단위로 교차 사용해 스크롤 시 구간을 구분한다(별도 구분선 없이도 리듬감 형성).
- 시맨틱 컬러는 관리자 화면(상태 뱃지)에만 사용하고, 공개 페이지 디자인에는 사용하지 않는다(공개 페이지는 브랜드 컬러로 통일성 유지).

## 2. 폰트 (Typography)

### 2-1. 서체

- **Pretendard** (국문/영문/숫자 가변폭 대응, 무료 라이선스, 국내 B2B/금융권에서 신뢰성 있는 산세리프로 널리 채택)
- 대체 폰트(fallback): `-apple-system, "Apple SD Gothic Neo", system-ui, sans-serif`
- 장식적 서체(필기체, 얇은 라이트체 단독 사용)는 사용하지 않는다 — 가독성과 격식을 동시에 떨어뜨림

### 2-2. 굵기(Weight) 사용 규칙

| 굵기     | 값  | 용도                          |
| -------- | --- | ----------------------------- |
| Bold     | 700 | H1~H2, 숫자 강조(시공실적 등) |
| SemiBold | 600 | H3~H4, 버튼 텍스트, 카드 제목 |
| Medium   | 500 | 강조 본문, 메뉴 텍스트        |
| Regular  | 400 | 본문, 설명문                  |

- Light(300) 이하 굵기는 **사용하지 않는다**: 얇은 글씨는 화면에서 흐릿하게 보여 40~60대 가독성을 해치고, "가벼운/임시적인" 인상을 줄 수 있다.

### 2-3. 타입 스케일

| 스타일                  | Desktop         | Mobile           | 줄간격 | 자간    |
| ----------------------- | --------------- | ---------------- | ------ | ------- |
| H1 (페이지 타이틀/Hero) | 40px / 2.5rem   | 28px / 1.75rem   | 1.3    | -0.02em |
| H2 (섹션 타이틀)        | 32px / 2rem     | 24px / 1.5rem    | 1.35   | -0.01em |
| H3 (서브 섹션)          | 24px / 1.5rem   | 20px / 1.25rem   | 1.4    | 0       |
| H4 (카드 제목)          | 20px / 1.25rem  | 18px / 1.125rem  | 1.4    | 0       |
| Body Large (본문 기본)  | 18px / 1.125rem | 16px / 1rem      | 1.7    | 0       |
| Body (보조 본문)        | 16px / 1rem     | 15px / 0.9375rem | 1.6    | 0       |
| Caption (캡션/Footer)   | 14px / 0.875rem | 13px / 0.8125rem | 1.5    | 0       |

**근거**: 일반적인 웹 가이드(본문 14~16px)보다 한 단계 큰 18px를 본문 기본값으로 사용한다. 40~60대 사용자의 평균적인 시력 저하를 고려한 의도적 선택이며, 줄간격(1.6~1.7)도 넉넉히 주어 장문(회사소개, 시공사례 설명)도 편하게 읽히도록 한다.

### 2-4. 숫자/통계 표기

- USP 카드의 숫자(시공실적 N건 등)는 Body Large보다 큰 별도 스타일(예: 36~48px, Bold)로 표기해 신뢰 지표를 시각적으로 강조한다.

## 3. 버튼 (Button)

### 3-1. 크기

| 크기   | 높이 | 패딩(좌우) | 폰트                  | 용도                             |
| ------ | ---- | ---------- | --------------------- | -------------------------------- |
| Large  | 52px | 28px       | Body Large / SemiBold | Hero CTA, 문의 제출 버튼         |
| Medium | 44px | 20px       | Body / SemiBold       | 일반 액션(더보기, 카드 내 버튼)  |
| Small  | 36px | 14px       | Caption / Medium      | 관리자 테이블 내 액션(수정/삭제) |

- **모든 버튼 높이는 최소 36px 이상**으로 고정한다(터치 타겟 44px 권장 기준을 Medium/Large에서 충족, Small은 관리자 데스크톱 전용이므로 예외 허용).

### 3-2. Variant

| Variant            | 배경                                       | 텍스트        | 보더                     | 용도                                |
| ------------------ | ------------------------------------------ | ------------- | ------------------------ | ----------------------------------- |
| Primary            | `brand-700`                                | `white`       | 없음                     | 일반 주요 액션(더보기, 로그인)      |
| CTA(Accent)        | `accent-500`(기본) → hover 시 `accent-600` | `white`       | 없음                     | 문의하기/견적문의 등 전환 핵심 버튼 |
| Secondary(Outline) | `white`                                    | `brand-700`   | 1.5px `brand-700`        | 보조 액션(취소, 사례보기)           |
| Ghost(Text)        | 투명                                       | `brand-700`   | 없음, hover 시 underline | 텍스트 링크형 버튼("전체보기 →")    |
| Disabled           | `neutral-200`                              | `neutral-400` | 없음                     | 동의 미체크 등 비활성 상태          |

### 3-3. 형태 규칙

- `border-radius: 8px` 고정 — pill(완전 둥근) 형태는 사용하지 않는다(과도하게 둥글면 가벼운 느낌, 직각이면 차가운 느낌이므로 중간값 채택)
- 그림자 없음(평면형). 입체적 그라데이션 버튼 금지 — 절제된 느낌 유지
- Hover: 배경색을 한 단계 어둡게 — `brand-700`→`brand-900`, `accent-500`→`accent-600`(1장에 정의된 토큰만 사용, 별도 다크 토큰 신설 금지) + `transition 180ms ease-out`
- Focus(키보드 접근): `outline 2px accent-600`, `outline-offset 2px`
- 아이콘 버튼(전화 클릭콜 등): 아이콘 + 텍스트 병기 필수(아이콘 단독 사용 금지 — 40~60대 사용자는 아이콘만으로 기능 추측이 어려울 수 있음)

## 4. 카드 (Card)

### 4-1. 공통 스펙

- 배경: `white`
- 보더: `1px solid neutral-200`
- 모서리: `border-radius: 12px`
- 그림자: `0 1px 3px rgba(15, 28, 48, 0.08)` (기본), hover 시 `0 6px 16px rgba(15, 28, 48, 0.12)`
- 패딩: Desktop `24px` / Mobile `16px`
- 그림자는 항상 절제된 농도로 사용한다(`opacity 0.08~0.12` 범위 고정) — 진한 그림자나 네온 글로우는 사용하지 않음

### 4-2. 시공사례 카드 (`WorkCard`)

- 구조: 썸네일(16:9, `object-cover`) → 카테고리 뱃지(`accent-600` 배경, 흰 텍스트, 작은 pill) → 제목(H4) → 간단설명(Body, 2줄 ellipsis) → 등록일(Caption, `neutral-400`)
- Hover: 썸네일 `scale(1.03)`, 카드 그림자 강화, `transition 200ms ease-out` — 둘 다 동시에 과하지 않게 적용

### 4-3. 정보/신뢰 카드 (사업자등록증, 인증서, 회사개요 표)

- 일반 카드보다 보더를 한 단계 진하게(`neutral-200` → 필요 시 `brand-500` 1px) 사용해 "공식 문서"라는 인상을 강조할 수 있음
- 사업자등록증/인증서 카드는 클릭 시 라이트박스 확대 — 카드 우상단에 작은 "확대" 아이콘 + 텍스트 배치

### 4-4. 통계(USP) 카드

- 아이콘(상단) + 숫자(Bold, 36~48px, `brand-700`) + 라벨(Body, `neutral-600`)
- 배경은 `neutral-50` 또는 `white` 단색, 그라데이션/일러스트 배경 금지(과한 장식은 전문성 인상을 약화시킴)

## 5. 여백 규칙 (Spacing)

### 5-1. 기본 단위

8px 그리드 기준, 4px 보정 단위 허용:
`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96 / 128 (px)`

### 5-2. 적용 규칙

| 영역                            | Desktop  | Mobile  |
| ------------------------------- | -------- | ------- |
| 섹션 간 수직 여백(예: Hero→USP) | 96~120px | 56~64px |
| 섹션 내부 상하 패딩             | 80px     | 48px    |
| 카드 grid gap                   | 24px     | 16px    |
| 카드 내부 패딩                  | 24px     | 16px    |
| 텍스트 블록 간(제목-본문)       | 16px     | 12px    |
| 폼 필드 간                      | 20px     | 16px    |
| 컨테이너 최대 너비              | 1280px   | 100%    |
| 컨테이너 좌우 패딩              | 24px     | 16px    |

**근거**: 섹션 간 여백을 의도적으로 넓게(96px+) 잡아 "정돈된 대기업/공식 사이트"의 느낌을 만든다. 여백이 좁으면 정보가 빽빽해 보여 영세업체 인상을 줄 수 있다는 점을 경계한다.

### 5-3. 정렬 규칙

- 모든 텍스트 블록은 좌측 정렬을 기본으로 한다(Hero 등 일부 중앙정렬 예외). 중앙정렬 과다 사용은 격식 있는 정보 전달에는 비효율적이다.
- 표(회사개요, 프로젝트 정보)는 라벨-값을 명확히 구분하는 2열 그리드로 정렬, 라벨 폭은 고정값(예: 120px)으로 통일한다.

## 6. 반응형 규칙 (Responsive)

### 6-1. 브레이크포인트 (`ARCHITECTURE.md`와 동일 기준)

| 구분         | 범위        | Tailwind 접두사     |
| ------------ | ----------- | ------------------- |
| Mobile       | 360~767px   | (기본, 접두사 없음) |
| Tablet       | 768~1023px  | `md:`               |
| Desktop      | 1024~1279px | `lg:`               |
| Wide Desktop | 1280px+     | `xl:`               |

### 6-2. 레이아웃 전환 규칙

| 요소                     | Mobile                       | Tablet                      | Desktop                            |
| ------------------------ | ---------------------------- | --------------------------- | ---------------------------------- |
| 카드 그리드(시공사례 등) | 1열(또는 가로 스와이프)      | 2열                         | 3~4열                              |
| Header 메뉴              | 햄버거 메뉴                  | 햄버거 메뉴(또는 일부 노출) | 전체 GNB 노출                      |
| 회사개요 표              | 라벨-값 세로 스택            | 2열 표                      | 2열 표                             |
| 사업분야 이미지+텍스트   | 이미지 위, 텍스트 아래(스택) | 좌우 교차 배치              | 좌우 교차 배치                     |
| 폼 필드                  | 풀와이드 1열                 | 풀와이드 1열                | 일부 필드 2열(이름/연락처 등) 허용 |
| 프로세스 다이어그램      | 세로 스텝                    | 세로 스텝                   | 가로 스텝                          |

### 6-3. 공통 규칙

- 터치 타겟(버튼, 링크, 체크박스): 모바일에서 **최소 44×44px** 확보
- 본문 폰트는 모바일에서 1단계 축소(타입 스케일표 참고)하되 16px 미만으로는 내려가지 않음(`Body Large` 모바일 16px가 하한선)
- 이미지: 모든 이미지는 `next/image` + `sizes` 속성으로 디바이스별 최적 해상도 제공, 모바일 Hero는 가로형 원본을 세로 크롭한 별도 버전 사용 권장
- 가로 스크롤(카테고리 탭, 카드 캐러셀)은 스크롤 가능 영역임을 인지시키기 위해 다음 항목이 화면 끝에서 일부(약 10~15%) 잘려 보이도록 배치(완전히 한 화면에 맞추지 않음)

## 7. 애니메이션 규칙 (Motion)

### 7-1. 기본 원칙

**"움직임은 최소한으로, 의미가 있을 때만"** — 이 사이트의 모션 목적은 화려함이 아니라 *상태 변화를 자연스럽게 인지시키는 것*이다. 트렌디한 패럴랙스, 3D 틸트, 글리치 효과, 통통 튀는(bounce) 이징은 사용하지 않는다(스타트업/캐주얼 브랜드 인상을 주어 신뢰감·안정감 목표와 충돌).

### 7-2. 타이밍/이징 표준값

| 용도                      | duration                | easing                                                          |
| ------------------------- | ----------------------- | --------------------------------------------------------------- |
| Hover(버튼, 카드, 링크)   | 150~200ms               | `ease-out`                                                      |
| 모달/라이트박스 열림·닫힘 | 200~250ms               | `ease-in-out`                                                   |
| 슬라이드 메뉴(모바일)     | 250ms                   | `ease-out` (열림) / `ease-in` (닫힘)                            |
| 스크롤 리빌(섹션 등장)    | 300~400ms               | `ease-out`, `translateY(8px)→0` + `opacity 0→1` 정도로 미세하게 |
| Hero 슬라이더 전환        | 500~600ms               | `ease-in-out`, fade 또는 slide 중 1가지만 사용(혼합 금지)       |
| 토스트(문의 접수 완료 등) | 200ms 등장 / 200ms 퇴장 | `ease-out`                                                      |

### 7-3. 적용 범위

- **사용하는 곳**: 버튼/카드 hover, 모바일 메뉴 열림, 모달/라이트박스, Hero 슬라이더 자동 전환(5~6초 간격), 스크롤 등장 효과(섹션당 1회, 과도한 stagger 금지)
- **사용하지 않는 곳**: 텍스트 타이핑 효과, 무한 반복 강조 애니메이션(깜빡임, 흔들림), 마우스 따라다니는 커스텀 커서, 배경 동영상/파티클 효과
- 스크롤 리빌은 섹션당 한 번만 발생시키고, 카드 여러 개가 동시에 stagger로 나타날 경우 간격은 최대 50~80ms로 짧게 제한(느린 순차 등장은 "로딩이 느리다"는 인상을 줄 수 있음)

### 7-4. 접근성

- `prefers-reduced-motion: reduce` 미디어쿼리 감지 시 모든 트랜지션/애니메이션을 즉시 전환(duration 0)으로 대체
- 자동 재생되는 Hero 슬라이더에는 일시정지 버튼 또는 hover 시 자동정지 적용(스크린리더/저시력 사용자 고려)

### 7-5. 로딩 상태

- 스피너는 단순한 원형 회전 애니메이션(`1ms~1.2s` 1회전, 무한 반복)만 사용
- 콘텐츠 로딩 중에는 화려한 스켈레톤 그라데이션 애니메이션 대신 단색 `neutral-100` 블록 + 미세한 opacity pulse(1.5s 주기)만 적용
