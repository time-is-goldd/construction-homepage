# 이미지 자산 매핑 (customer-assets → public/images)

이 문서는 `customer-assets/`에 제공된 고객 원본 자료를 `public/images/`로 정리한 내역과,
WIREFRAME.md 기준 각 페이지/섹션에서 어떤 이미지를 사용할지에 대한 매핑을 기록한다.

> 이번 단계는 **자산 정리(이동/정규화/매핑 문서화)만** 수행했다. 회사명·사업분야 카테고리·
> 페이지 문구 등 콘텐츠를 `문구.txt` 및 실제 시공사례에 맞춰 정렬하는 작업은 별도 Phase에서 진행한다.
> 이미지 압축/리사이즈(최적화)도 원본 손상 우려가 있어 이번 단계에서는 실행하지 않고 대상만 표시했다.

## 3. 파일명 정규화 규칙

- 케이스 폴더명: 한글 표기 → 영문 kebab-case 슬러그 (예: `농협 신림` → `nonghyup-sillim`)로 변경.
  공백·괄호가 포함된 원본 경로는 URL/코드에서 다루기 번거롭고, 기존 `lib/mock-works.ts`의
  `categorySlug` 표기 방식과도 일관성을 맞췄다.
- 파일명: 케이스별로 `01, 02, 03 …` 일련번호 + 소문자 확장자(`jpeg → jpg` 통일)로 변경.
- **바이트 단위 완전 중복 파일은 제거**했다 (MD5 해시 비교). 예: `이화축산` 폴더의 `1.jpg~7.jpg`는
  같은 폴더의 `KakaoTalk_...` 원본 9개와 내용이 완전히 동일해 제외했고, `(1)` 접미사가 붙은
  4개 파일도 원본과 동일해 제외했다. (152장 원본 → 139장으로 정리)
- "모두 이동" 방침에 따라 대표 이미지(썸네일) 선정은 하지 않았다 — 각 폴더의 `01.*`을
  잠정 대표 이미지로 가정했으나, 실제 콘텐츠 작업 시 재검토가 필요하다.

## 5. 누락된 자료 (요청 필요)

1. **인증서/자격증 이미지** — 사업자등록증, ISO 인증서 등. 홈 5번 섹션, 회사개요 3번 섹션,
   보유역량 3번 섹션에서 모두 필요하나 전혀 제공되지 않았다.
2. **회사소개용 인물/사옥 사진** — 대표 인사말 섹션의 대표이미지, 조직도/인력현황 섹션 자료.
   둘 다 WIREFRAME상 "(선택)" 항목이라 텍스트만으로 진행 가능하지만, 있으면 페이지 완성도가 높아진다.
3. **보유 장비 단독 컷** — 크레인/굴착기 등 장비를 단독으로 촬영한 사진. 시공사례 사진에 장비가
   부분적으로 등장하지만 보유역량 페이지용으로 쓰기엔 적합하지 않다.
4. **메인 비주얼 추가 컷** — Hero 슬라이더는 최대 5장 구성인데 현재 1장(`hero-aerial-01.png`,
   1405×567)뿐이며, 이 해상도도 풀블리드 배경으로 쓰기엔 낮은 편이다(아래 6번 참고).
5. **시공사례 공사 유형 확인** — `시공사례/농장/` 하위 7개 케이스(농협 신림/의성/합천,
   돈돈팜 곡성/횡성, 이화축산, 진도 효돈)는 폴더명에 공사 유형(신축/리모델링/순환시설/환기)이
   명시되어 있지 않다. `신축 공사`/`리모델링 공사` 폴더처럼 유형이 명확한 5건과 달리, 이 7건은
   실제 사업분야 카테고리에 매핑하려면 고객 확인이 필요하다.

## 6. 이미지 최적화 후보 (실행은 하지 않음, 표시만)

| 대상 | 현재 상태 | 권장 처리 |
|---|---|---|
| `works/ihwa-livestock-fattening-renovation/*` (57장, 총 233MB) | 휴대폰 원본 4000×2250, 장당 4~5MB | 웹용 리사이즈(긴 변 1600px 내외) + WebP/AVIF 변환 시 90% 이상 용량 절감 예상 |
| `works/andong-piglet-house-construction/*`, `andong-fattening-house-reconstruction/*`, `andong-sow-house-reconstruction/*`, `hwaseong-farm-reconstruction/*` (총 140MB) | PNG 스크린샷/원본 다수 | 사진 성격 콘텐츠는 PNG보다 JPEG/WebP가 훨씬 작음 — 포맷 변경 권장 |
| `logo/logo-full*.{png,jpg}` | 6332~6486px 폭의 인쇄용 해상도 | 웹에서는 실제 표시 폭(약 150~240px) 기준 2~3배수인 ~720px면 충분 — 대폭 축소 가능 |
| `hero/hero-aerial-01.png` | 1405×567, PNG | Hero 배경(풀블리드)으로 쓰려면 오히려 **더 큰 원본**이 필요. 현재 해상도로는 1920px 이상 화면에서 확대 시 흐려짐 — 고객에게 고해상도 원본 재요청 권장. 보유 해상도 그대로 쓴다면 JPEG/WebP 변환만으로도 용량 절감 가능 |

`public/images` 전체 용량은 현재 **약 411MB**다. Next.js `<Image>` 컴포넌트가 런타임에 자동
리사이즈/포맷 변환을 해주지만, 원본 자체가 git 저장소에 그대로 들어가면 클론/배포 속도에
영향을 준다. 실제 최적화(리사이즈·포맷 변환) 실행은 원본 손상 우려가 있어 별도 작업으로
진행하는 것을 권장한다.
